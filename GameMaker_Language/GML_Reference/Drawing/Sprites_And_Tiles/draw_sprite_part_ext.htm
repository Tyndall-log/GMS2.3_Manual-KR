draw_sprite_part_ext

이 함수는 draw_sprite_part () 와 동일한 규칙에 따라 지정된 위치에서 선택한 스프라이트의 일부를 그립니다. 이제 만 파트를 스케일링하거나 색상을 혼합하거나 화면에 그릴 때 알파를 변경할 수 있습니다. draw_sprite_ext () 로 스프라이트를 그릴 때와 동일합니다. 텍스처 페이지에서 자동 자르기를 허용하면 텍스처 페이지를 만들기 위해 스프라이트 주변의 추가 "빈"공간이 제거되므로이 기능이 예상대로 작동하지 않을 수 있습니다. 이 문제를 해결하려면 텍스처 그룹 편집기 에서 텍스처 페이지 설정을 설정하여 자동 자르기 를 비활성화해야 합니다 .

참고 : 이 기능을 사용하여 그릴 때 스프라이트 x 오프셋 및 y 오프셋 은 무시되고 룸의 지정된 x / y 위치에서 왼쪽 상단 모서리와 함께 스프라이트 부분이 그려집니다.

참고 :이 함수는 비트 맵 스프라이트에만 유용하며 SWF 또는 JSON (스파인) 스프라이트에서는 작동하지 않습니다.

참고 : 색상 혼합은 WebGL이 활성화 된 경우 HTML5 대상에만 권장되지만 혼합 색상이 활성화되어 있지 않으면 여전히 색상을 설정하여 스프라이트를 정상적으로 혼합합니다. 그러나 이러한 방식으로 모든 블렌딩을 수행하면 복제 스프라이트가 생성되어 캐시에 저장되고 필요할 때 사용됩니다. 이것은 최적의 상태와는 거리가 멀고 여러 색상 변경을 사용하면 WebGL을 활성화하지 않으면 게임 성능이 저하됩니다. WebGL을 사용하지 않으려는 경우 sprite_set_cache_size () 함수를 사용하여 필요한 경우 글꼴 캐시 크기를 설정하여이를 제한 할 수 있습니다.

 

구문:

draw_sprite_part_ext(sprite, subimg, left, top, width, height, x, y, xscale, yscale, colour, alpha);

논의 기술
스프라이트 그릴 스프라이트의 인덱스입니다.
Subimg 그릴 스프라이트의 하위 이미지 (프레임)입니다 (image_index 또는 -1은 객체의 현재 애니메이션 프레임과 상관이 있습니다).
왼쪽 그릴 영역의 왼쪽 상단 모서리에있는 스프라이트의 x 위치입니다.
상단 그릴 영역의 왼쪽 상단 모서리에있는 스프라이트의 y 위치입니다.
폭 그릴 영역의 너비입니다.
신장 그릴 영역의 높이입니다.
엑스 스프라이트를 그릴 위치의 x 좌표입니다.
와이 스프라이트를 그릴 위치의 y 좌표입니다.
xscale 승수로서 스프라이트의 수평 스케일링 : 1 = 일반 스케일링, 0.5는 절반 등 ...
yscale 승수와 같은 스프라이트의 수직 스케일링 : 1 = 일반 스케일링, 0.5는 절반 등 ...
색깔 스프라이트를 혼합 할 색상입니다. c_white는 정상적으로 표시하는 것입니다.
알파 스프라이트의 알파 (0에서 1까지 0, 1은 불투명)

 

보고:

N/A

 

예:

draw_sprite_part_ext(sprite_index, image_index, 8, 8, sprite_width-16, sprite_height-16, x, y, 2, 0.5, c_black, 1);

그러면 스프라이트 (sprite_index)에 할당 된 인스턴스와 현재 애니메이션 프레임 (image_index)이 그려 지지만 스프라이트의 4면에서 8px의 여백이 줄어 듭니다. 그런 다음 평소 너비를 두 배로 늘리고 평소 높이의 절반으로 늘리고 알파는 여전히 1이지만 검은 색과 혼합되어 실루엣으로 바뀝니다.

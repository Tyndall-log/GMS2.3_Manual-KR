gpu_set_blendmode_ext

GameMaker Studio 2가 픽셀을 그리면 소스 색상 (그린 픽셀의 색상)과 대상 색상 (이미 그리는 픽셀에있는 색상)이 있으므로 픽셀의 새로운 색상, 소스 및 대상 색상은 선택한 블렌드 모드에 따라 계산됩니다. 색상의 각 구성 요소는 0과 1 사이의 부동 소수점 값으로 저장되며, 새로운 색상은 소스 색상의 각 구성 요소에 몇 가지 요소를 곱하고 대상 색상의 각 구성 요소에 다른 요소를 곱한 다음 구성 요소별로 결과를 함께 제공합니다.

이 기능을 통해 사용자 정의 블렌드 모드를 생성하기 위해 함께 고려해야하는 다양한 컴포넌트 파트를 설정할 수 있습니다. 소스 및 대상은 모두 빨강, 녹색, 파랑 및 알파 구성 요소를 가지며 다음 차트에서 소스의 RGBA는 (Rs, Gs, Bs, As)로 간주되고 대상은 (Rd, Gd, Bd, Ad)로 간주됩니다. 사용 가능한 11 개의 상수는이 함수에서 사용될 때 소스 또는 대상 (또는 둘 다) 일 수 있습니다.

일정한 블렌드 팩터 (빨강, 녹색, 파랑, 알파)
bm_zero (0, 0, 0, 0)
bm_one (1, 1, 1, 1)
bm_src_colour (Rs, Gs, Bs, As)
bm_inv_src_colour (1-R, 1-G, 1-B, 1-A)
bm_src_alpha (As, As, As, As)
bm_inv_src_alpha (1-As, 1-As, 1-As, 1-As)
bm_dest_alpha (광고, 광고, 광고, 광고)
bm_inv_dest_alpha (1- 광고, 1- 광고, 1- 광고, 1- 광고)
bm_dest_colour (Rd, Gd, Bd, Ad)
bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad)
bm_src_alpha_sat (f, f, f, 1) 여기서 f = min (As, 1-Ad)

 

이 함수에 두 개의 개별 인수를 제공하거나 인수 배열을 제공 할 수 있습니다 ( 예 : gpu_get_blendmode_ext () 함수에 의해 리턴 됨). 배열을 제공하는 경우 다음 두 가지 요소가 있어야합니다.

  • [0] = 소스 블렌드 모드 (기본값은 bm_src_alpha )
  • [1] = 대상 블렌드 모드 (기본값은 bm_inv_src_alpha )

이러한 서로 다른 블렌드 모드의 작동 방식을 시각화하는 것은 상당히 어려울 수 있으므로 아래에서 단일 배경에 대해 여러 조합의 블렌드 모드 조합으로 그려지는 단일 스프라이트의 기본 상호 작용을 보여주는 간단한 차트를 찾을 수 있습니다. 사용하는 블렌드 모드의 실제 결과는 스프라이트의 색상, 스프라이트의 알파 값 및 배경의 색상 및 알파에 따라 크게 달라집니다.

중대한! WebGL을 사용하지 않는 HTML5는 다음 모드를 올바르게 표시 할 수 없습니다 .

  • bm_src_colour
  • bm_inv_src_colour
  • bm_dest_colour
  • bm_inv_dest_colour
  • bm_src_alpha_sat

 

구문:

gpu_set_blendmode(src, dest);

논의 기술
src 소스 블렌드 모드 계수 (위 상수 참조).
대상 대상 혼합 모드 계수 (위 상수 참조)

 

반환값:

N/A

 

예:

gpu_set_blendmode_ext(bm_src_alpha, bm_one);
draw_circle_colour(100, 100, 50, c_white, c_black, 0);
gpu_set_blendmode(bm_normal);

이렇게하면 검은 색이 투명 해져서 흰색이 바깥 쪽이 강하고 원 중심에서 점차 약해지면서 '빛나는'효과가 나타납니다. 원을 그린 후에는 블렌드 모드가 재설정되어 추가 블렌딩이 그 이후의 모든 것에 적용되지 않도록합니다.

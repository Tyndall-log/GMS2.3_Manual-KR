steam_ugc_create_query_all_ex

이 함수는 UGC 데이터베이스를 쿼리하는 데 사용할 수 있습니다. 이 함수에는 아이템을 소비 할 게임의 ID 및 / 또는 아이템을 생성 한 게임의 ID가 필요하며, 생성 할 쿼리 유형, 쿼리 할 항목 유형에 대해 다음 상수를 사용해야합니다. 그리고 퀘스트 할 페이지 번호. 쿼리는 최대 50 개의 항목을 반환합니다.

 

쿼리 유형 상수 기술
ugc_query_RankedByVote Steam 설명서를 참조하십시오.
ugc_query_RankedByPublicationDate -
ugc_query_AcceptedForGameRankedByAcceptanceDate -
ugc_query_RankedByTrend -
ugc_query_FavoritedByFriendsRankedByPublicationDate -
ugc_query_CreatedByFriendsRankedByPublicationDate -
ugc_query_RankedByNumTimesReported -
ugc_query_CreatedByFollowedUsersRankedByPublicationDate -
ugc_query_NotYetRated -
ugc_query_RankedByTotalVotesAsc -
ugc_query_RankedByVotesUp -
ugc_query_RankedByTextSearch -

 

일치 유형 상수 기술
ugc_match_Items Steam 설명서를 참조하십시오.
ugc_match_Items_Mtx -
ugc_match_Items_ReadyToUse -
ugc_match_Collections -
ugc_match_Artwork -
ugc_match_Videos -
ugc_match_Screenshots -
ugc_match_AllGuides -
ugc_match_WebGuides -
ugc_match_IntegratedGuides -
ugc_match_UsableInGame -
ugc_match_ControllerBindings -

이 함수는 다른 쿼리 함수에서 사용하기 위해 변수에 저장해야하는 고유 한 쿼리 핸들 값을 반환합니다. 이 함수는 쿼리 만 준비하지만 실제로는 전송하지 않습니다. steam_ugc_send_query () 함수를 호출해야합니다. 더 자세한 steam_ugc_query _ * () 함수를 사용하여 검색 요청을 실제로 보내기 전에 세분화 할 수 있습니다.

 

구문:

steam_ugc_create_query_all_ex(query_type, match_type, page, creator_app_id, consumer_app_id);

논의 기술
query_type 생성 할 데이터 쿼리 유형입니다 (아래에 나열된 상수 참조).
match_type 쿼리 할 UGC 항목의 유형입니다 (아래에 나열된 상수 참조).
페이지 조회 할 페이지 번호입니다.
creator_app_id 아이템 작성자 앱 ID
consumer_app_id 소비자 앱 ID

 

보고:

Query Handle

 

예:

query_handle = steam_ugc_create_query_all_ex(ugc_query_RankedByVote, page, global.AccountID, 0, global.GameID);

위의 코드는 쿼리 요청을 생성하고 나중에 사용할 수 있도록 핸들에 변수를 저장합니다.

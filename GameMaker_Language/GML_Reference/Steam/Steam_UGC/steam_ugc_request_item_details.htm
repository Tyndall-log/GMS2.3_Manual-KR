steam_ugc_request_item_details

이 기능은 주어진 파일 ID에 대한 정보를 검색하는 데 사용할 수 있습니다. 파일 ID를 제공하고 확인할 최대 보존 기간을 제공하면 (자세한 내용은 Steam 문서 참조)이 함수는 Steam Async 이벤트 에서 세부 정보를 올바르게 확인하는 데 사용할 수있는 고유 한 비동기 ID 값을 반환합니다. 이 이벤트에는 async_load DS 맵이 있으며 다음 키 / 값 쌍으로 채워집니다.

  • " id "-이벤트를 트리거 한 함수가 리턴 한 비동기 ID
  • " result "-작업 결과 (실제 값). 이것은 GML 상수 ugc_result_success 또는 다른 실수입니다. 따라서이 상수를 확인하여 호출이 성공했는지, 그렇지 않은 경우 정상적으로 작동하지 않았는지 확인해야합니다. 반환 된 나머지 가능한 값은 Steam "EResult"값의 결과로 표시되며 가능한 89 개의 모든 값에 대한 SDK 헤더에 steamclientpublic.h 가 표시됩니다.
  • " event_type "-문자열 " ugc_item_details "
  • " cached_data "-리턴 된 세부 사항이 로컬 캐시에서 온 경우 true 또는 서버에서 가져온 경우 false
  • " published_file_id" "-항목의 고유 한 공개 파일 ID를 보유합니다.
  • " file_type "-사용 된 파일 유형
  • " creator_app_id "-항목 작성자의 Steam ID
  • " consumer_app_id "-품목 소비자의 스팀 ID
  • " title "-항목의 제목
  • " description "-항목에 대한 설명
  • " steam_id_owner "-아이템 소유자의 스팀 ID
  • " time_created "-항목이 처음 작성된 시간
  • " time_updated "-항목이 마지막으로 업데이트 된 시간
  • " time_added_to_user_list "-항목이 구독 된 시간
  • " 가시성 "- 항목의 가시성 (참조 steam_ugc_set_item_visibility () 반환 된 상수)
  • " 금지 "-품목의 금지 여부 ( true ) 또는 아닌지 ( false )
  • " accept_for_use "-항목의 사용이 승인되었는지 ( true ) 아닌지 ( false )
  • " tags_truncated "-짧은 버전의 태그 배열
  • " tags "-항목의 태그 배열
  • " handle_file "-항목의 고유 한 파일 핸들
  • " handle_preview_file "-항목의 이미지 미리보기에 대한 고유 한 핸들 ( 미리보기 이미지를 다운로드하기 위해 steam_ugc_download () 를 사용하여 인수로 사용할 수 있음)
  • " filename "-아이템 파일의 이름
  • " file_size "-아이템 파일의 크기
  • " preview_file_size "-미리보기 이미지의 크기
  • " url "-상품의 전체 URL
  • " votes_up "-받은 투표 수
  • " votes_down "-받은 다운 투표 수
  • " score "-상품의 전체 점수
  • " account_id_owner " -Steam ID 소유자의 계정 ID ( steam_ugc_create_query_user_ex () 함수에서 사용할 수 있음)

 

통사론:

steam_ugc_request_item_details(published_file_id, max_age_seconds);

논의 기술
publish_file_id 확인할 UGC의 고유 파일 ID입니다.
max_age_seconds 확인할 데이터의 "나이"(30-60 초 권장)

확장 된 예 :

이 예에서는 항목에 대한 세부 정보 요청을 보낸 다음 결과 async_load DS 맵을 구문 분석하여 일부 변수를 설정합니다. 먼저 요청을 보냅니다.

steam_details = steam_ugc_request_item_details(global.fileID, 60);

위의 코드는 전역 변수에 파일 ID가 저장된 항목에 대한 세부 정보를 요청하고 반환 된 정보와 함께 Steam Async 이벤트를 트리거합니다. 이 경우 맵을 구문 분석하고 일부 값을 변수에 저장하여 사용자에게 정보를 표시하는 데 사용할 수 있습니다.

var map_id = async_load[? "id"];
var result = async_load[? "result"];
if (map_id == steam_details) && (result == ugc_result_success)
    {
    mTitle = async_load[? "title"];
    mDesc = async_load[? "description"];
    mTags = async_load[? "tags"];
    m_hPreviewFile = async_load[? "handle_preview_file"];
    m_hOwnerSteamId = async_load[? "steam_id_owner"];
    mOwnerAccountId = async_load[? "account_id_owner"];
    mPubFileId = async_load[? "published_file_id"];
    mScore = async_load[? "score"];
    }

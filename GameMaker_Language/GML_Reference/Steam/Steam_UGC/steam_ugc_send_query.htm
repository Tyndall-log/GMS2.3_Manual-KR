steam_ugc_send_query

이 함수는 쿼리 요청을 보내는 데 사용할 수 있습니다. 먼저 "쿼리 핸들"을 반환하는 적절한 steam_ugc_create_query _ * () 함수를 사용하여 쿼리를 정의합니다. 이 핸들은이 함수에서 쿼리 요청을 보내기 전에 필터 등을 설정하는 데 사용됩니다. 이 함수는 Steam Async 이벤트 에서 세부 정보를 올바르게 확인하는 데 사용할 수있는 고유 한 비동기 ID 값을 반환합니다. 이 이벤트에는 async_load DS 맵이 있으며 다음 키 / 값 쌍으로 채워집니다.

  • " id "-이벤트를 트리거 한 함수가 리턴 한 비동기 ID
  • " result "-작업 결과 (실제 값). 이것은 GML 상수 ugc_result_success 또는 다른 실수입니다. 따라서이 상수를 확인하여 호출이 성공했는지, 그렇지 않은 경우 정상적으로 작동하지 않았는지 확인해야합니다. 반환 된 나머지 가능한 값은 Steam "EResult"값의 결과로 표시되며 가능한 89 개의 모든 값에 대한 SDK 헤더에 steamclientpublic.h 가 표시됩니다.
  • " event_type "-문자열 " ugc_query "
  • " num_results "-반환 된 결과 수 (최대 50)
  • " total_matching "-일치하는 총 결과 수
  • " cached_data "-이 데이터가 로컬 온 디스크 캐시에서 검색되었는지 여부를 나타냅니다 ( true 또는 false ).
  • " results_list "-DS 목록 색인 색인. 각 목록 항목은 특정 항목의 세부 사항을 포함하는 DS 맵 색인입니다.

비동기 이벤트 유형 "ugc_query"가 표시되면 결과 목록을 구문 분석하고 각 DS 맵에서 다음 정보를 추출 할 수 있습니다 (목록의 항목 당 하나의 맵).

  • " published_file_id" "-항목의 고유 한 공개 파일 ID를 보유합니다.
  • " file_type "-사용 된 파일 유형
  • " creator_app_id "-항목 작성자의 Steam ID
  • " consumer_app_id "-품목 소비자의 스팀 ID
  • " title "-항목의 제목
  • " description "-항목에 대한 설명
  • " steam_id_owner "-아이템 소유자의 스팀 ID
  • " time_created "-항목이 처음 작성된 시간
  • " time_updated "-항목이 마지막으로 업데이트 된 시간
  • " time_added_to_user_list "-항목이 구독 된 시간
  • " 가시성 "- 항목의 가시성 (참조 steam_ugc_set_item_visibility () 반환 된 상수)
  • " 금지 "-품목의 금지 여부 ( true ) 또는 아닌지 ( false )
  • " accept_for_use "-항목의 사용이 승인되었는지 ( true ) 아닌지 ( false )
  • " tags_truncated "-짧은 버전의 태그 배열
  • " tags "-항목에 대한 태그가 포함 된 문자열이며 각 태그는 쉼표로 구분됩니다
  • " handle_file "-항목의 고유 한 파일 핸들
  • " handle_preview_file "-항목의 이미지 미리보기에 대한 고유 한 핸들 ( 미리보기 이미지를 다운로드하기 위해 steam_ugc_download () 를 사용하여 인수로 사용할 수 있음)
  • " filename "-아이템 파일의 이름
  • " file_size "-아이템 파일의 크기
  • " preview_file_size "-미리보기 이미지의 크기
  • " url "-상품의 전체 URL
  • " votes_up "-받은 투표 수
  • " votes_down "-받은 다운 투표 수
  • " score "-상품의 전체 점수
  • " account_id_owner " -Steam ID 소유자의 계정 ID ( steam_ugc_create_query_user_ex () 함수에서 사용할 수 있음)

 

구문:

steam_ugc_send_query(ugc_query_handle);

논의 기술
ugc_query_handle 전송할 쿼리 핸들

 

반환값:

Boolean

 

예:

var query_handle = steam_ugc_create_query_all(ugc_query_RankedByTrend, ugc_match_Items, 1);
steam_ugc_query_add_required_tag(query_handle, "RPG");
steam_ugc_query_set_return_long_description(query_handle, true);
steam_ugc_query_set_allow_cached_response(query_handle, true);
query_ID = steam_ugc_send_query(query_handle);

위의 코드는 쿼리 요청을 만들고 쿼리를 보내기 전에 쿼리 요청을 추가로 정의하는 나머지 함수에서 나중에 사용할 수 있도록 핸들을 로컬 변수에 저장합니다.

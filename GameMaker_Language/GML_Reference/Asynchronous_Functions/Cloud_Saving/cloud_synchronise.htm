cloud_synchronise

이 함수는 일반적으로 새 게임을 시작할 때 호출되며 게임을 시작할 때 클라우드 서비스의 현재 상태를 검색하는 데 사용됩니다. 이 함수는 고유 한 id 값을 반환 한 다음 비동기 클라우드 이벤트 에서 생성 된 DS 맵에서 관련 정보를 검색하는 데 사용됩니다.

이 기능은 데이터를 클라우드로 전송 한 다음 적절한 비동기 이벤트를 트리거합니다. 이 경우 반환 된 async_load DS 맵에서 다음 값을 확인할 수 있습니다.

  • "상태": 상태 코드를 보유하며, 음수 값은 오류를 나타내고 설명은 "errorString"에 포함됩니다. 0 값 (또는 양수 값)은 성공 (정확한 값은 아래 참조)을 나타내며 "resultString"은 반환 된 데이터 또는 성공 메시지를 포함합니다.
  • "id": 호출 된 함수에서 반환 된 id 일련의 cloud_ 요청을 발생시키는 경우 응답을받는 요청을 알아야 하므로이 값을 사용하여 원래 요청을 보냈을 때 저장 한 값과 비교하여 올바른 요청을 찾습니다.
  • "설명": 마지막으로 업로드 된 파일에 대한 설명입니다.
  • "resultString": 클라우드에서 리턴 된 데이터 블로 브 인 문자열을 보유합니다.
  • "errorString": 오류에 대한 오류 문자열을 반환합니다.

반환 된 "상태"맵 항목의 정확한 의미는 다음 표에 설명되어 있습니다.

 

상태 값 errorString / resultString 기술
-1 errorString = "<SERVICE>에 로그인하지 않았습니다" 주어진 클라우드 서비스에 성공적으로 로그인하지 않았습니다
0 resultString = 복구 된 데이터 클라우드에서 다운로드 한 새로운 게임 데이터 (cloud_synchronise 호출에 따름)
1 resultString = "AlreadySynchronized" 마지막으로 cloud_synchronise를 호출 한 이후 새 데이터가 없습니다.
2 resultString = "ConflictDeferral" 충돌이 발생했지만 게이머는이를 무시하기로 선택했습니다.
삼 resultString = "GameUploadSuccess" cloud_string_save () 또는 cloud_file_save ()의 데이터가 클라우드에 성공적으로 업로드되었습니다
-엔 errorString = 오류 설명 다른 음수는 동기화 실패를 의미합니다

 

통사론:

cloud_synchronise();

확장 된 예 :

이 기능은 게임 시작 이벤트 와 같은 이벤트 또는 게임 의 첫 번째 방에있는 객체에서 호출됩니다. 클라우드 서버에서 현재 데이터 블로 브가 최신인지 확인하기 위해 날짜 또는 아닙니다.

cloud_check = cloud_synchronise();

그런 다음 비동기 클라우드 이벤트 에서 반환 된 DS Map 을 확인하여 상태 및 반환 된 문자열 (있는 경우)을 확인하려면 다음 코드를 사용합니다.

if ds_map_find_value(async_load, "id") == cloud_check
   {
   if ds_map_find_value(async_load, "status") < 0
      {
      show_message_async("Cloud Services not available. Please check connectivity.");
      }
   else
      {
      if ds_map_find_value(async_load, "status") == 0
         {
         var file = file_text_open_write("Save.txt");
         file_text_write_string(file, ds_map_find_value(async_load, "resultString"));
         file_text_close(file);
         }
      }
   }

위의 코드는 올바른 비동기 함수 호출이 수정되고 있는지 확인한 다음 반환 된 클라우드 DS 맵의 상태를 가져옵니다. 상태가 음수이면, 문제가 발생하여 사용자에게 알립니다. 그렇지 않으면 코드에서 계속 동기화 된 데이터를 검색하여 나중에 사용할 수 있도록 텍스트 파일에 씁니다.

achievement_load_leaderboard

이 기능은 주어진 리더 보드에 대한 정보를 서버에 요청합니다. 리더 보드는 게임의 개발자 대시 보드에서 이전에 생성되어 있어야하며 제공된 ID는 함수의 첫 번째 인수로 제공해야합니다. 관련 키 / 값 쌍으로 채워진 async_load 맵이 포함 된 콜백 소셜 비동기 이벤트 를 트리거합니다. 이 DS 맵 의 id 키는 올바른 콜백을 식별하는 데 사용되며 (어떤 주어진 비동기 이벤트에 대해 두 개 이상의 트리거 기능이있을 수 있음) 상수 achievement_leaderboard_info 및 다른 여러 키 / 값 쌍과 쌍을 이룹니다 각 플레이어. 지도의 정확한 내용은 다음과 같습니다.

참고 :이 기능은 iOS 전용입니다.

  • " id "-이 함수의 경우 achievement_leaderboard_info
  • " leaderboardid "-제공자 대시 보드에 정의 된 리더 보드의 고유 ID입니다.
  • " numentries "-리더 보드에서받은 항목 수입니다.
  • " PlayerN "-플레이어의 이름입니다. 여기서 "N"은 순위표 항목 목록 내 위치에 해당하는 정수 값입니다.
  • " PlayeridN "-플레이어의 고유 사용자 ID "N"
  • " RankN "-순위표 내 플레이어 "N"의 순위입니다.
  • " ScoreN "-플레이어 "N"의 점수입니다.

다음과 같은 필터 상수를 사용하여 친구 나 모든 플레이어가 게시 한 점수 만 검색하도록 결과를 필터링하도록 요청하고 최소 및 최대 색인 값을 제공하여 리더 보드의 정확한 부분을 표시 할 수 있습니다.

 

일정한 기술
achievement_filter_friends_only 플레이어 친구의 점수 만 얻으십시오.
achievement_filter_all_players 모든 플레이어의 모든 점수를 얻으십시오.

 

 

통사론:

achievement_load_leaderboard(ident, minindex, maxindex, filter)

논의 기술
신분 개발자 대시 보드에 표시된 리더 보드의 고유 ID입니다.
최소 색인 리더 보드 데이터를 가져올 시작 색인 값입니다.
maxindex 리더 보드 데이터를 가져올 최대 색인 값입니다.
필터 친구 목록에있는 결과 또는 모든 플레이어에 대한 결과를 필터링할지 여부 (아래 설명 참조)

확장 된 예 :

다음 코드는 플레이어가 achievement_login () 을 사용하여 게임 계정에 로그인하여 Android 또는 iOS에서 게임의 모든 친구 순위표 목록을 가져온 후에 호출 될 수 있습니다.

if os_type == os_ios
    {
    achievement_load_leaderboard("CgkIs9_51u0PEAIQBw", 1, 100, achievement_filter_friends_only);
    }

그러면 주어진 리더 보드에 대한 정보 요청이 전송되고 다음 데이터가 포함 된 특수 async_load ds_map을 사용하여 비동기 콜백이 생성됩니다.

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_leaderboard_info
    {
    var lbid = ds_map_find_value(async_load,"leaderboardid");
    if lbid == "CgkIs9_51u0PEAIQBw" || lbid == "leaderboard1id"
        {
        global.numentries = ds_map_find_value(async_load,"numentries");
        for(var i = 0; i < numentries; i++;)
            {
            global.playername[i] = ds_map_find_value(async_load, "Player" + string(i));
            global.playerid[i] = ds_map_find_value(async_load, "Playerid" + string(i));
            global.playerrank[i] = ds_map_find_value(async_load, "Rank" + string(i));
            global.playerscore[i] = ds_map_find_value(async_load, "Score" + string(i));
            achievement_get_pic(global.playerid[i]);
            }
        }
    }

위의 코드는 소셜 비동기 이벤트 에서 반환 된 DS Map 을 확인하고 "id"가 확인되는 상수와 일치하면 전송되는 리더 보드가 원래 함수 호출과 일치하는지 확인하여 맵을 반복하고 모든 여러 배열에서 다른 값. 또한 맵의 각 항목에 대한 이미지를 요청합니다 ( achievement_get_pic () 참조).

achievement_send_challenge

이 기능은 네트워크를 통해 선택한 플레이어에게 챌린지를 보냅니다. achievement_load_friends () 또는 achievement_load_leaderboard () 함수를 사용하여 플레이어 ID 를 얻을 수 있으며 iTunes Connect 또는 Google Play 대시 보드에서 도전 과제를 만들 때 주어진 도전 과제 인 고유 값인 challengeid 도 제공해야합니다. 또한 챌린지 유형 을 설정하고 점수와 짧은 문자 메시지를 제공해야 합니다 . 이것은 다음 상수 중 하나 일 수 있습니다.

 

일정한 기술
achievement_type_score_challenge 점수 값에 따른 챌린지.
achievement_type_achievement_challenge 업적에 따른 도전.

 

이 함수는 챌린지를받을 플레이어의 콜백 소셜 비동기 이벤트 를 트리거하며,이 경우 변수 async_load 에서 ds_map을 참조하게됩니다. 이 DS 맵 의 id 키는 올바른 콜백을 식별하는 데 사용되며 (어떤 주어진 비동기 이벤트에 대해 두 개 이상의 트리거 기능이있을 수 있음) 상수 업적 _challenge_received 및 각 플레이어. 지도의 정확한 내용은 다음과 같습니다.

참고 : 사용자는 수신 된 챌린지에 대한 토스트 알림을받을 수도 있지만, achievement_show_challenge_notifications () 함수를 사용하여 선택한 값에 따라 달라집니다.

  • " id "-이 함수의 경우 achievement_leaderboard_info
  • " playerid "-챌린지의 플레이어 ID입니다.
  • " issuerid "-챌린지의 발급자 ID.
  • " state "- 유효하지 않은 , 보류 중 , 완료 또는 거부 된 경우 0-3 ( 문자열) 값을 갖는 챌린지 상태입니다.
  • " message "-챌린지에 대한 문자 메시지.
  • " completedate "-챌린지 완료 날짜입니다.
  • " 발행일 "-이의 제기 발행일.
  • " type "-주어진 챌린지 유형. 위에 나열된 두 상수 중 하나 일 수 있습니다.
  • " identifier "-제공자 대시 보드에 설정된 챌린지의 식별 문자열
  • " score "-챌린지와 관련된 점수입니다 (해당되는 경우).

 

통사론:

achievement_send_challenge(playerid, challengeid, score, type, message)

논의 기술
playerid 도전 할 플레이어의 고유 ID입니다.
challengeid 제공자 챌린지에있는 고유 한 챌린지 ID.
점수 이길 점수.
유형 발행 할 챌린지 유형-아래에 주어진 두 상수 중 하나.
메시지 챌린지에 첨부 할 메시지입니다.

 

반환값:

N/A

 

확장 된 예 :

네트워크를 통해 챌린지를 보내려면 다음 코드가 필요합니다.

achievement_send_challenge(global.playerid[0], global.challengeid[0], score, achievement_type_score_challenge, "Beat that sucker!");

이 요청은 챌린지가 지시 된 플레이어의 게임에서 소셜 이벤트 를 트리거하며 다음과 같은 방식으로 처리 될 수 있습니다.

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_challenge_received
    {
    player_id = ds_map_find_value(async_load, "playerid");
    issuer_id = ds_map_find_value(async_load, "issuerid");
    state = ds_map_find_value(async_load, "state");
    message = ds_map_find_value(async_load, "message");
    date_completed = ds_map_find_value(async_load, "completeddate");
    date_issued = ds_map_find_value(async_load, "issueddate");
    ach_type = ds_map_find_value(async_load, "type");
    ach_ident = ds_map_find_value(async_load, "identifier");
    ach_score = ds_map_find_value(async_load, "score");
    }

위의 코드는 소셜 비동기 이벤트 에서 리턴 된 DS 맵 을 확인하고 "id"가 점검중인 상수와 일치하는 경우 맵의 각 키에 대한 관련 값을 추출하여 나중에 사용할 수 있도록 변수에 저장합니다.

achievement_get_challenges

이 함수는 모든 현재 챌린지에 대한 정보를 서버에 요청하고 관련 키 / 값 쌍으로 채워진 async_load 맵을 포함하는 콜백 소셜 비동기 이벤트 를 트리거합니다. 이 DS 맵 의 id 키는 올바른 콜백을 식별하는 데 사용되며 (어떤 주어진 비동기 이벤트에 대해 하나 이상의 트리거 기능이있을 수 있음) 상수 achievement_challenge_list_received 및 다른 여러 키 / 값 쌍과 쌍을 이룹니다 각 도전. 지도의 정확한 내용은 다음과 같습니다.

참고 :이 기능은 iOS 전용입니다.

  • " id "-이 기능의 경우 achievement_challenge_list_received 이어야합니다
  • " numchallenges "-현재 사용 가능한 챌린지 수 (로컬 및 원격).
  • " ChallengeNplayerid "- 챌린지 의 플레이어 ID입니다. 여기서 "N"은 정수입니다. EG : "Challenge5playerid"는 목록에서 다섯 번째 챌린지의 플레이어 ID입니다.
  • " ChallengeNissuerid "- 챌린지 를 발행 한 사람의 ID입니다. 여기서 "N"은 정수입니다. EG : "Challenge2issuerid"는 목록에서 두 번째 챌린지의 발급자 ID입니다.
  • " ChallengeNstate "-챌린지 "N"의 상태로, 유효하지 않거나 보류 중이 거나 완료 되었거나 거부 된 경우 0-3 ( 문자열) 값을 갖습니다.
  • " ChallengeNmessage "-챌린지 "N"에 대한 문자 메시지.
  • " ChallengeNissueddate "- "N"에 대한 발급 날짜입니다.
  • " ChallengeNcompleteddate "-챌린지 "N"의 완료 날짜입니다.
  • " ChallengeNtype "-주어진 챌린지 유형. 두 상수 중 하나 일 수 있습니다.
일정한 기술
achievement_filter_friends_only 플레이어 친구의 점수 만 얻으십시오.
achievement_filter_all_players 모든 플레이어의 모든 점수를 얻으십시오.

 

  • " ChallengeNidentifier "- 챌린지 의 식별 문자열
  • " ChallengeNscore "- 챌린지 와 관련된 점수입니다.

확장 된 예 :

플레이어가 achievement_login ()을 사용하여 게임 계정에 로그인 한 후 다음 코드가 호출 될 수 있습니다.

achievement_get_challenges();

현재 챌린지 정보에 대한 요청을 전송하고 다음 데이터를 포함하는 특수 async_load ds_map을 사용하여 비동기 콜백을 생성합니다.

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_challenge_list_received
   {
   var numentries = ds_map_find_value(async_load,"numchallenges");
   for(var i = 0; i < numentries; i++;)
      {
      player_id[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"playerid");
      issuer_id[i] = ds_map_find_value(async_load, "Challenge" + string(i) +"issuerid");
      state[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"state");
      message[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"message");
      date_completed[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"completeddate");
      date_issued[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"issueddate");
      ach_type[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"type");
      ach_ident[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"identifier");
      ach_score[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"score");
   }

위의 코드는 Social Asynchronous Event 에서 반환 된 DS Map 을 확인하고 "id"가 점검중인 상수와 일치하면 여러 배열에있는 모든 다른 값을 저장하는 맵을 반복합니다. 그런 다음이 정보를 사용하여 게임에서 나만의 맞춤형 도전 페이지를 만들 수 있습니다.

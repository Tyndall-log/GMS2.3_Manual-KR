http_post_string

컴퓨팅에서, 포스트 요청은 클라이언트가 파일을 업로드하거나 완성 된 양식을 제출할 때와 같이 검색 요청의 일부로 서버에 데이터를 보내야 할 때 사용되며 GameMaker Studio 2의 경우에도 동일합니다. 받는 반면 HTTP_GET () 만을 URL은 (서버, http_post_string로 전송 요청 방법)도 서버에 송신되어 새로운 자원의 생성 또는 기존 자원의 갱신 또는 될 수있는 문자열을 포함 양자 모두. 이 함수를 사용할 때 HTTP 요청 매개 변수 (웹 페이지에서 양식을 제출할 때 때때로 URL의 끝에 "고정 된 비트")를 사용할 수 있습니다.

이 이벤트는 모든 HTTP 이벤트에 의해 선택되는 "콜백"을 생성합니다.이 경우이 이벤트 전용의 DS 맵 (일반적으로 "사전"이라고 함)이 생성되고 특수 변수에 저장됩니다. async_load 이 ds_map은 리턴되는 데이터의 유무에 따라 다른 값을 포함합니다. 예를 들어 파일을 요청한 경우 각 데이터 패킷이 수신 될 때 이벤트가 여러 번 트리거되어 진행률 표시 줄을 표시 할 수 있습니다. ds_map의 일반적인 구조는 다음과 같습니다.

  • id : 명령에서 반환 된 ID입니다. 일련의 http_ 요청을 발생시키는 경우 응답을받는 요청을 알아야 하므로이 값을 사용하여 원래 요청을 보낼 때 저장 한 값과 비교하여 올바른 요청을 찾습니다.
  • status : 오류의 경우 0, 완료의 경우 0, 패킷 수신의 경우 1보다 작은 값을 반환합니다 (자세한 내용은 아래 참조).
  • 결과 : 수신 된 데이터 (문자열 만 해당)
  • url : 요청한 전체 URL입니다.
  • http_status : 원시 http 상태 코드 (사용 가능한 경우). 이것은 대부분의 브라우저에서 표준 웹 상태 코드를 반환합니다.

게임에 여러 개의 패킷이 반환되면 콜백 "상태"키는 1을 반환하며,이 경우 DS 맵에는 다음과 같은 추가 키가 있습니다.

  • "contentLength": 웹 서버가 수신해야한다고 말한 파일의 크기입니다 (서버가이 데이터를 반환하지 않으면 -1 일 수 있음).
  • "sizeDownloaded": 이미 다운로드 된 데이터의 크기입니다.

참고 : 브라우저의 XSS 보호로 인해 도메인 간 리소스 요청 및로드 시도가 차단되어 빈 결과가 반환 될 수 있습니다. 자세한 내용은 도메인 간 문제 부분을 ​​참조하십시오.

 

구문:

http_post_string(url, string);

논의 기술
url 정보를 얻으려는 서버의 웹 주소
끈 지정된 URL로 보내려는 문자열

 

반환값:

Real

 

확장 된 예 :

http_post_string () 함수는 모든 이벤트에서 호출 할 수 있으며, 비동기이기 때문에 콜백은 거의 즉각적이거나 몇 초가 걸릴 수 있습니다. 함수를 호출하는 것은 간단하며 다음과 같습니다.

var str = "name=" + global.player_name + "&score=" + string(global.player_score);
post = http_post_string("http://www.angusgames.com/game?game_id=" + string(global.game_id), str);

위의 코드는 변수 str에 저장된 추가 데이터 (URL 인코딩 형식의 플레이어 이름 및 점수)와 함께 지정된 매개 변수를 사용하여 검색 요청을 지정된 URL로 보냅니다. 이 정의 된 모든 비동기 트리거 HTTP를 이벤트 콜백이 수신되면, 당신은 다양한 http_post_string () 함수를 사용해야 당신은 당신이 올바른 코드를 실행할 수 있는지 확인하기 위해 가변 포스트에 저장된에 대해 반환 된 "ID"를 확인할 수 있습니다. 다음 예제 코드는이를 수행하는 방법을 보여줍니다.

var r_str = "null";
if ds_map_find_value(async_load, "id") == post
    {
    if ds_map_find_value(async_load, "status") == 0
        {
        r_str = ds_map_find_value(async_load, "result");
        }
    }

위 코드는 먼저 생성 된 DS 맵의 ID를 확인한 다음 콜백 상태를 확인합니다. 값이 0 (신호 성공)과 같으면 콜백의 결과는 나중에 사용하기 위해 변수에 저장되며, 그렇지 않으면 변수는 기본값 (이 경우 "null")을 유지합니다.

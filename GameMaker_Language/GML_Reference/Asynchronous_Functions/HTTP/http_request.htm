http_request

이 기능을 사용하면 HTTP 헤더 요청을 작성하여 HTTP 트랜잭션의 작동 매개 변수를 정의 할 수 있습니다. 이는 RESTful API를 사용하는 경우 HTTP 헤더를 통한 인증과 같은 여러 가지에 사용할 수 있습니다. 이 기능을 사용하려면 요청하는 서버의 전체 IP 주소와 요청 유형 (문자열, 아래 참고 참조)이 필요합니다 : "GET", "HEAD", "POST", "PUT", " 삭제 ","TRACE ","OPTIONS "또는"CONNECT ". 키 / 값 쌍의 DS 맵 (문자열, 키는 헤더 필드, 값은 헤더에 필요한 데이터)을 제공해야하며 마지막 인수는 추가 할 수있는 선택적 데이터 문자열입니다. 필요하지 않은 경우 0 또는 빈 문자열 ""일 수 있습니다. 버퍼를 보낼 수도 있습니다 (자세한 내용은 버퍼 섹션 참조).이 경우 마지막 인수는 보낼 버퍼의 "핸들"입니다.

참고 : HTTP 헤더는 일반적으로 "key : value"형식을 따르지만 GMS는 ds_map에서 이러한 쌍을 생성하므로 맵 키 또는 값 문자열에 콜론 ":"을 포함 할 필요가 없습니다.

참고 : body 인수에 버퍼를 사용할 때 버퍼 탐색 위치가 시작 (0)에 있으면 본문이 전송되지 않고 http 호출의 응답으로 버퍼가 채워지지만 버퍼 탐색 위치가 아닌 경우 0이면 버퍼 문자열 내용이 본문으로 전송됩니다.

참고 : 브라우저의 XSS 보호로 인해 도메인 간 리소스 요청 및로드 시도가 차단되어 빈 결과가 반환 될 수 있습니다. 자세한 내용은 도메인 간 문제 부분을 ​​참조하십시오.

이 이벤트는 모든 HTTP 이벤트에 의해 선택되는 "콜백"을 생성합니다.이 경우이 이벤트 전용의 DS 맵 (일반적으로 "사전")이 생성되고 특수 변수에 저장됩니다. async_load 이 DS 맵에는 요청 기능과 관련된 다음 두 개의 키가 있습니다.

  • id : 명령에서 반환 된 ID입니다. 일련의 http_ 요청을 발생시키는 경우 응답을받는 요청을 알아야 하므로이 값을 사용하여 원래 요청을 보낼 때 저장 한 값과 비교하여 올바른 요청을 찾습니다.
  • response_headers : 값이 0보다 크거나 같으면 HTTP 요청에 대한 응답으로 리턴 된 HTTP 헤더가 포함 된 ds_map을 보유합니다.

 

구문:

http_request(url, method, header_map, body);

논의 기술
url 정보를 얻으려는 서버의 웹 주소
방법 요청 방법 (일반적으로 "POST"또는 "GET"이지만 모든 방법이 지원됨)
header_map 필수 헤더 필드가있는 ds_map
몸 헤더를 따라 전송되는 데이터 (이진 버퍼 핸들 일 수 있음)

 

반환값:

Real

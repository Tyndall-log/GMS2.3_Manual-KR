physics_joint_prismatic_create

회전 조인트와 마찬가지로 프리즘 조인트는 1 자유도 만 갖지만이 조인트를 사용하면 회전이 아닌 축을 기준으로 방향이 지정되며 실제로 모든 형태의 회전을 방지합니다. 작동 방식을 시각화하는 데 도움이되는 이미지는 다음과 같습니다.

우리는 앵커 포인트에 두 개의 인스턴스가 결합되어 있습니다. 벡터에서 물리 세계의 (0,0) 좌표에 대한 두 개의 w_axis x / y 좌표로 정의 된 이동 축이 있습니다. 1 (0,1)은 조인트를 수직 프리즘 조인트로 만듭니다). 이 조인트는 스프링이나 피스톤처럼이 축을 기준으로 만 움직일 수 있습니다. 하한 또는 상한 트랜스 한계를 설정하면 기본적으로이 축을 따라 이동량이 제한됩니다. 여기서 0 위치는 w_anchor x / y로 정의한 정확한 지점이므로 음수 값은 "왼쪽"으로 이동합니다. 다음 다이어그램에 표시된대로 축을 따라 해당 점의 양수 값과 "오른쪽"의 양수 값 ( "왼쪽"과 "오른쪽"이 상대적인 용어임을 인식하십시오!) :

관절에 모터가 있는지 여부를 정의 할 수도 있습니다. 이는 충돌에 영향을받지 않으면 조인트가 축을 따라 한 방향으로 움직입니다. 이 방향은 모터 속도로 정의되며 양수는 축 좌표를 향하고 ( "오른쪽"), 음수는 기준점을 향합니다 ( "왼쪽"). "max_motor_force"인수는 이동 속도를 제한하여 지속적으로 가속되는 모터를 얻지 못하고 충돌이 이동에 미칠 수있는 영향을 제한합니다. 이런 방식으로 조인트 모터를 사용하여 조인트 속도를 0으로 설정하고 최대 힘을 ​​작지만 중요한 값으로 설정하여 조인트 마찰을 시뮬레이션 할 수 있습니다. 모터는 조인트가 움직이지 않도록 시도하지만 상당한 하중을 발생시킵니다.

모든 관절과 마찬가지로 "col"값을 true 로 설정하면 두 인스턴스가 서로 상호 작용하고 충돌 할 수 있지만 충돌 이벤트가있는 경우 에만 충돌이 발생합니다. 그러나 false 로 설정되어 있으면 충돌이 발생하지 않습니다. .

 

구문:

physics_joint_prismatic_create(inst1, inst2, w_anchor_x, w_anchor_y, w_axis_x, w_axis_x, lower_trans_limit, upper_trans_limit, limit, max_motor_force, motor_speed, motor, col)

논의 기술
inst1 조인트와 연결하는 첫 번째 인스턴스
inst2 조인트와 연결할 두 번째 인스턴스
w_anchor_x 게임 세계 내에서 조인트가 고정 된 x 좌표
w_anchor_y 게임 세계 내에서 조인트가 고정 된 y 좌표
w_axis_x 축 벡터의 x 성분
w_axis_y 축 벡터의 y 성분
lower_trans_limit 관절 운동에 대한 허용 된 하한
upper_trans_limit 관절 운동에 대한 허용 된 하한
한도 관절의 움직임을 제한할지 (true) 아닌지 (false)
max_motor_force 모터의 최대 이동 속도를 설정합니다
motor_speed 이것은 모터가 움직여야하는 속도입니다
모터 모터의 활성화 여부 (true) 또는 아닌지 (false)
안부 두 인스턴스가 충돌 할 수 있는지 (true) 아닌지 (false)

 

반환값:

Integer (index of the joint)

 

예:

var mainFixture, o_id;
mainFixture = physics_fixture_create();
physics_fixture_set_circle_shape(mainFixture, sprite_get_width(sprite_index) / 2);
o_id = instance_create_layer(x+25, y, "Instances", obj_Piston);
physics_fixture_bind(mainFixture, id);
physics_fixture_bind(mainFixture, o_id);
physics_joint_prismatic_create(id, o_id, x, y, 0, 10, 0, 0, 0, 5, 0, 1, 1);
physics_fixture_delete(mainFixture);

위의 코드는 새로운 조명기를 생성하고 정의한 다음 생성 된 조명기를 두 개의 새로운 객체에 바인딩하여 "obj_Piston"인스턴스를 생성합니다. 그런 다음 첫 번째 인스턴스와 동일한 x / y 좌표에서 앵커 위치와 x / y 위치의 벡터와 x / y 축 (이 경우 0,10, "다운"입니다. 이 축을 따라 이동하는 양에는 제한이 없지만 관절 마찰을 시뮬레이션하기 위해 속도가 0이고 최대 힘이 5 인 모터를 추가했습니다.

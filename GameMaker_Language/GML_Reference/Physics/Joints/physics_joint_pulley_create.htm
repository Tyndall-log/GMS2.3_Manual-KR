physics_joint_pulley_create

풀리 조인트는 하나의 움직임이 다른 하나의 움직임에 직접 영향을 미치는 방식으로 물리 세계에서 두 인스턴스를 연결하는 데 사용됩니다. 이 조인트는 먼저 월드 공간에 두 지점으로 고정되며, 각 지점은 질량 중심의 인스턴스에 연결됩니다. 인스턴스의 원점을 기준으로 로컬 앵커 x / y 좌표를 설정하여 변경할 수 있습니다. 즉, 인스턴스와 물리 세계 앵커 포인트로의 실제 연결은 원점 이외의 다른 위치에서 오프셋 될 수 있습니다. 다음 이미지는이를 보여줍니다.

inst2를 보면 로컬 앵커 포인트가 원점의 "왼쪽"으로 20 픽셀 오프셋 된 것을 볼 수 있습니다 (월드 앵커 포인트는 룸 좌표를 사용하여 정의되고 로컬 앵커 포인트는 인스턴스의 출처). 풀리 조인트의 비율을 지정하여 한쪽 또는 다른 쪽이 더 빨리 움직 이도록 지시 할 수 있습니다.이 값은 1보다 작 으면 첫 번째 인스턴스가 더 빨리 움직이고 (EG 0.5가 두 배 빠르게 움직입니다) 1보다 큰 값이됩니다. 두 번째 인스턴스는 더 빠르게 움직입니다 (EG 2는 두 배 빠르게 움직입니다).

모든 관절과 마찬가지로 "col"값을 true 로 설정하면 두 인스턴스가 서로 상호 작용하고 충돌 할 수 있지만 충돌 이벤트가있는 경우 에만 충돌이 발생합니다. 그러나 false 로 설정되어 있으면 충돌이 발생하지 않습니다. .

 

구문:

physics_joint_pulley_create(inst1, inst2, w_anchor1_x, w_anchor1_y, w_anchor2_x, w_anchor2_y, l_anchor1_x, l_anchor1_y, l_anchor2_x, l_anchor2_y, ratio, col)

논의 기술
inst1 조인트와 연결하는 첫 번째 인스턴스
inst2 조인트와 연결할 두 번째 인스턴스
w_anchor1_x 게임 세계 내에서 조인트의 첫 번째 부분이 고정되는 x 좌표
w_anchor1_y 게임 세계 내에서 조인트의 첫 번째 부분이 고정 된 y 좌표
w_anchor2_x 게임 세계 내에서 조인트의 두 번째 부분이 고정 된 x 좌표
w_anchor2_y 게임 세계 내에서 조인트의 두 번째 부분이 고정 된 y 좌표
l_anchor1_x 조인트 1이 첫 번째 인스턴스에 고정 된 로컬 x 좌표
l_anchor1_y 조인트 1이 첫 번째 인스턴스에 고정 된 로컬 y 좌표
l_anchor2_x 조인트 2가 두 번째 인스턴스에 고정 된 로컬 x 좌표
l_anchor2_y 조인트 2가 두 번째 인스턴스에 고정 된 로컬 y 좌표
비율 두 인스턴스 간 속도 비율 설정
안부 두 인스턴스가 충돌 할 수 있는지 (true) 아닌지 (false)

 

반환값:

Integer (index of the joint)

 

예:

var t_fix, inst1, inst2;
t_fix = physics_fixture_create();
physics_fixture_set_circle_shape(t_fix, 20);
inst1 = instance_create_layer(150, room_height - 90, "Instances", obj_Block);
inst2 = instance_create_layer(300, room_height - 90, "Instances", obj_block);
physics_fixture_bind(t_fix, inst1);
physics_fixture_bind(t_fix, inst2);
physics_joint_pulley_create(inst1, inst2, 150, room_height - 140, 300, room_height - 140, 0, 0, 0, 0, 2, 0);
physics_fixture_delete(t_fix);

위의 코드는 새로운 조명기를 생성하고 정의하고 "obj_Block"의 두 인스턴스를 생성 한 다음이 조명기를 두 개의 바인딩합니다. 그런 다음 조인트에 대한 오프셋없이 2 : 1의 비율로 이러한 인스턴스간에 풀리 조인트를 정의합니다 (즉, inst2가 더 빠르게 움직일 것임을 의미). 풀리의 두 인스턴스간에 충돌이 발생하지 않습니다.

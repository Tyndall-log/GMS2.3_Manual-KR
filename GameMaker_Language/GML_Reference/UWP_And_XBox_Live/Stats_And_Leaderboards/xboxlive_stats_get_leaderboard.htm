xboxlive_stats_get_leaderboard

이 함수는 주어진 통계에 대한 순위의 글로벌 순위표를 검색하는 데 사용할 수 있습니다. 사용자 ID (예 : xboxlive_get_user () 함수에 의해 리턴 됨 ), 통계 문자열 ( "추천 통계"로 등록 할 때 정의 된대로)을 제공 한 다음 리더 보드 정보에 대한 세부 사항을 지정합니다. 검색하고 싶습니다. int 또는 real 통계에 대한 글로벌 리더 보드 만 검색 할 수 있지만 문자열 통계에 대해서는 검색 할 수 없습니다 .

중대한! 글로벌 리더 보드에서 사용되는 통계는 XDP / Windows Dev Center에서 "추천 통계"로 등록되어야합니다. 그렇지 않으면 오류가 리턴됩니다. 로컬 (소셜) 리더 보드를 원하면 xboxlive_stats_get_social_leaderboard () 함수를 참조하십시오.

이 함수는 콜백을 생성하여 시스템 비동기 이벤트를 트리거합니다. 이 이벤트에는 내장 DS 맵 async_load 가 있으며 다음 키에 대해 구문 분석해야합니다.

  • " id "-일정한 achievement_stat_event를 유지합니다
  • " event "- " GetLeaderboardComplete "문자열을 보유합니다
  • " userid "-요청과 연관된 사용자 ID
  • " error "-성공하면 0, 오류가 있으면 다른 값. 다음 두 가지 가장 일반적인 오류가 반환됩니다. 2145844844 : 요청 된 리더 보드를 찾을 수 없습니다 (통계가 추천 통계로 등록되지 않았습니다)
  • -2145844848 : 잘못된 요청 (통계가 유효한 리더 보드 유형이 아닙니다 (예 : 문자열))
  • " errormessage "-사용 가능한 경우 오류 메시지가있는 문자열
  • " display_name "-제공자 대시 보드에 정의 된 리더 보드의 고유 ID입니다.
  • " numentries "-리더 보드에서받은 항목 수입니다.

    • DS 맵의 나머지 부분에는 다음 형식의 리더 보드 데이터도 포함됩니다 (여기서 "N"은 리더 보드 데이터에서 0에서 "numentries"까지의 위치 임).

    • " Player N "- 플레이어 의 이름입니다. 여기서 "N"은 순위표 항목 목록 내 위치에 해당하는 정수 값입니다.
    • " Playerid N "-플레이어의 고유 사용자 ID "N"
    • " Rank N "-순위표 내의 플레이어 "N"의 순위입니다.
    • " Score N "-플레이어 "N"의 점수입니다.

     

    통사론:

    xboxlive_stats_get_leaderboard(user_id, stat, num_entries, start_rank, start_at_user, ascending);

    논의 기술
    user_id 리더 보드를 가져올 사용자의 사용자 ID
    통계 전역 리더 보드를 생성하는 통계 (문자열)
    num_entries 글로벌 리더 보드에서 검색 할 항목 수
    시작 _ 순위 리더 보드에서 시작할 순위입니다 ( "start_at_user"인수가 true 로 설정된 경우 0 사용)
    start_at_user 사용자 ID 순위에서 시작하려면 true 로 설정하고 그렇지 않으면 false로 설정하십시오 ( "start_rank"인수가 0이 아닌 경우 false로 설정)
    오름차순 오름차순의 경우 true 로, 내림차순의 경우 false 로 설정하십시오.

    확장 된 예 :

    다음은이 기능을 사용하는 방법에 대한 확장 된 예입니다. 시작하려면 Room Start 또는 Create 같은 일부 이벤트에서 호출 하십시오 .

    xboxlive_stats_get_leaderboard(user_id, "GlobalTime", 20, 1, false, true);

    위 코드는 게임의 모든 소셜 리더 보드 위치 목록을 얻기 위해 호출되며 다음과 같은 방식으로 처리 할 소셜 비동기 이벤트 콜백을 생성합니다.

    if (async_load[? "id"] == achievement_stat_event)
        {
        if (async_load[? "event"] == "GetLeaderboardComplete")
            {
            global.numentries = async_load[? "numentries"];
            for(var i = 0; i < numentries; i++;)
                {
                global.playername[i] = async_load[? "Player" + string(i)];
                global.playerid[i] = async_load[? "Playerid" + string(i)];
                global.playerrank[i] = async_load[? "Rank" + string(i)];
                global.playerscore[i] = async_load[? "Score" + string(i)];
                }
            }
        }

    위의 코드는 소셜 비동기 이벤트에서 반환 된 DS 맵을 확인하고 "id"가 확인중인 상수와 일치하는 경우 맵을 반복하고 다른 모든 항목을 저장하기 전에 반환 된 리더 보드 데이터에 의해 이벤트가 트리거되었는지 확인합니다. 많은 전역 배열의 값.

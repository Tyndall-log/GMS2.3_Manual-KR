dot_product

내적은 두 벡터 간의 각도 관계를 나타내는 값으로, 두 벡터를 취하고이를 곱한 다음 결과를 추가하여 구합니다. "dot product"라는 이름은이 작업을 지정하는 데 자주 사용되는 중심 점 "Â"에서 파생됩니다 (대안 "scalar product"는 결과의 벡터 특성이 아니라 스칼라를 강조합니다).

실제 수학 공식은 다음과 같이 작성할 수 있습니다.

따라서 2D에서 벡터 a [x1, y1] 및 b [x2,2]의 내적 은 x1x2 + y1y2 이므로 GameMaker Studio 2의 내적은 다음과 같이 계산됩니다.

a · b = (x1*x2)+(y1*y2);

내적에 대한 흥미로운 점은 입력 벡터에 의해 형성된 각도와의 관계이며 다음과 같이 표현할 수 있습니다.

a · b = (length of a) * (length of b) * cos(angle)

즉, 두 벡터의 내적은 각 벡터의 길이를 곱한 벡터 간 각도의 코사인과 같습니다. 설명 할 이미지는 다음과 같습니다.

참고 : 벡터에 대한 자세한 내용은 여기를 참조 하십시오 .

우리는 그들의 내적 결과와 관련하여 두 벡터에서 진술 할 수있는 몇 가지가 있습니다 :

  • a와 b가 서로 수직 인 경우 (90도) cos (ang) 이 0이므로 내적 결과는 0이됩니다.
  • a와 b 사이의 각도가 90 도보 다 작 으면 cos (ang) 이 양수이므로 내적은 양수 (0보다 큼 ) 입니다.
  • a와 b 사이의 각도가 90 도보 다 크면 cos (ang) 이 음수이므로 내적은 음수 (0보다 작음 ) 입니다.

그렇다면 이것이 게임을하는 우리에게는 어떤 의미입니까? 글쎄,이 수학적 관계는 몇 가지 상황에서 사용될 수 있지만, 그것을 보는 가장 좋은 방법은 실제 시나리오를 구축하고 무슨 일이 일어나고 있는지 직접 보는 것입니다. 이 작업을 수행하는 가장 간단한 방법 중 하나는 플랫폼 게임에서 적에 대한 간단한 "시선"검사를 생성하는 것입니다. 벡터 법선 .

기본적으로, 우리는 적으로부터 벡터 법선을 가져와야합니다 (즉, 방향과 시야 거리로 구성됨) 그리고 플레이어의 벡터를 적에게 가져와야합니다. 그런 다음이 벡터의 dot_product를 얻습니다. 결과가 양수이면 플레이어가 표시되고 음수이면 표시되지 않습니다. 이 작업을 수행하기위한 실제 코드는 아래 예에서 제공됩니다.

 

통사론:

dot_product(x1, y1, x2, y2)

논의 기술
x1 첫 번째 벡터의 x 좌표입니다.
y1 첫 번째 벡터의 y 좌표입니다.
x2 두 번째 벡터의 x 좌표입니다.
y2 두 번째 벡터의 y 좌표입니다.

 

보고:

Real

 

예:

var x1, y1, x2, y2;
x1 = lengthdir_x(1, image_angle);
y1 = lengthdir_y(1, image_angle);
x2 = o_Player.x - x;
y2 = o_Player.y - y;
if dot_product(x1, y1, x2, y2) > 0 seen=true else seen=false;

위의 코드는 인스턴스 이미지 각도를 사용하여 벡터를 만든 다음 플레이어 객체 "o_Player"의 벡터를 가져옵니다. 마지막으로이 두 벡터의 내적을 계산하고 0보다 크면 변수 "seen"을 true로 설정하고 0보다 작거나 같으면 "false"로 설정합니다.

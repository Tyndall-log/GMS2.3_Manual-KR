buffer_create

이 함수를 사용하면 게임에서 메모리의 일부를 버퍼로 할당 할 수 있습니다.이 함수는 변수에 저장해야하고 버퍼에 대한 모든 추가 함수 호출에 사용해야하는 고유 버퍼 ID 를 반환합니다. 그런 다음 버퍼를 사용하여 다른 유형의 데이터를 저장할 수 있습니다 ( buffer_write () 함수를 사용하여 버퍼에 쓸 때 지정됨) . 다음 상수를 사용하여 버퍼 유형을 정의합니다.

 

일정한 기술
buffer_fixed 고정 크기의 버퍼.
buffer_grow 데이터가 추가 될 때 동적으로 "증가"하는 버퍼
buffer_wrap 데이터가 "랩핑"되는 버퍼입니다. 추가되는 데이터가 버퍼 크기의 한계에 도달하면 덮어 쓰기가 버퍼의 시작 부분에 다시 배치되고 이후부터 계속해서 쓰기가 계속됩니다.
buffer_fast 읽기 / 쓰기 속도가 매우 빠른 특수 "스트립 형"버퍼. buffer_u8 데이터 유형 에만 사용할 수 있으며 1 바이트로 정렬 되어야합니다 .
buffer_vbuffer 이 유형의 버퍼는 꼭짓점 버퍼로만 사용됩니다.

 

버퍼 유형 외에도 버퍼의 바이트 정렬 을 설정해야합니다. 이 값은 버퍼에 저장하려는 데이터에 따라 다르며 대부분의 경우 1 값은 완벽합니다. 그러나 일부 작업의 경우 특정 정렬이 필수적 이며 잘못 정렬하면 오류가 발생할 수 있습니다 (정렬에 대한 자세한 내용은 버퍼 참조). 다음은 각 데이터 유형에 가장 적합한 값을 보여주는 일반 안내서입니다 (의심 할 경우 1의 정렬 사용).

  • 문자열은 1 바이트로 정렬되어야합니다.
  • 부호있는 또는 부호없는 8 비트 정수는 임의의 값으로 정렬 될 수 있지만 빠른 버퍼 ( buffer_write () 참조)의 경우 1로 정렬 되어야합니다 .
  • 부호있는 또는 부호없는 16 비트 정수는 2 바이트로 정렬되어야합니다.
  • 부호있는 또는 부호없는 32 비트 정수는 4 바이트로 정렬되어야합니다.
  • 최대 16 비트의 부동 소수점은 2 바이트로 정렬되어야합니다. (현재 지원되지 않습니다!)
  • 최대 32 비트의 부동 소수점은 4 바이트로 정렬되어야합니다.
  • 최대 64 비트의 부동 소수점은 8 바이트로 정렬되어야합니다.

참고 : 잘못된 선택은 성능에 부정적인 영향을 줄 수 있으므로 바이트 정렬은 매우 중요합니다.

참고 : 메모리 누수를 방지하기 위해 더 이상 필요없는 메모리에서 이와 같이 동적으로 생성 된 리소스를 제거하는 것이 중요합니다. 따라서 생성 한 버퍼를 완료하면 buffer_delete ()를 사용하여 다시 해제해야합니다.

 

통사론:

buffer_create(size, type, alignment)

논의 기술
크기 버퍼의 크기 (바이트).
유형 작성할 버퍼의 유형입니다 (위의 상수 목록 참조).
조정 버퍼의 바이트 정렬

 

보고:

Int (Vertex buffer ID)

 

예:

player_buffer = buffer_create(16384, buffer_fixed, 2);

위 코드는 16384 바이트의 메모리를 버퍼에 할당하고 나중에 사용할 수 있도록 변수 "player_buffer"에 저장된 해당 버퍼의 인덱스를 반환합니다. 버퍼는 2 바이트 경계로 정렬됩니다.

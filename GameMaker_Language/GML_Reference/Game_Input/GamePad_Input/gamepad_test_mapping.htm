gamepad_test_mapping

이 기능은 그것을 허용하는 대상에 게임 패드 매핑을 설정하는데 사용될 수 있습니다. 게임 패드에 설정할 "슬롯"인덱스를 제공 한 후 다음 필드와 함께 SDL 형식을 사용하여 생성 한 맵 문자열을 제공합니다.

<guid>,<description>,platform:<platform-name>,<bindings>

각 필드에 포함되어야하는 세부 사항은 다음과 같습니다.

  • GUID를 - 고유 디바이스 타입을 식별하는 숫자와 문자의 스트링 (이는 사용하여 검색 될 수 gamepad_get_guid ()를 함수)
  • description- 장치의 ASCII 설명 ( gamepad_get_description () 함수를 사용하여 검색 할 수 있음)
  • platform- name- "platform : android", "platform : windows", "linux"또는 "mac" 이됩니다 ( 이 함수를 호출하면 문자열에 자동으로 추가 되므로 필요하지 않습니다. 자신의 매핑 문자열을 만들 때 제공하기 위해)
  • bindings- 실제 입력을 특정 GML 상수에 바인딩하기 위해 ','로 구분 된 항목 세트

쉼표로 구분 된 항목 목록은 각 항목에 대한 특정 형식을 따릅니다.

<abstract-gp-name>:<value-binding>

여기서 <값 바인딩> (즉, 직접 게임 패드 입력)은 다음 중 하나 일 수 있습니다.

  • a <N> -게임 패드의 축을 나타냅니다. 여기서 <N>은 0에서 gamepad_axis_count () -1까지의 축 중 하나입니다.
  • b <N> -게임 패드의 버튼을 나타냅니다. 여기서 <N>은 0에서 gamepad_button_count () -1까지의 버튼 중 하나입니다.
  • h <N>. <mask> -게임 패드의 모자를 나타냅니다. 여기서 <N>은 0에서 gamepad_hat_count () -1까지의 모자를 나타내고 <mask> 는 1, 2, 4 또는 8입니다. .

각 입력 값은 <> abstract-gp-name> 으로 표시되는 SDL 이름에 바인드되어야하며, 이는 차례로 GML 상수에 바인드됩니다. 아래 표는 SDL 이름과 GameMaker Studio 2 상수의 동등성을 보여줍니다.

매핑 문자열을 구성하면 다음과 같이 보일 것입니다.

"050000005e040000fd020000ffff3f00,Xbox Wireless Controller,a:b0,b:b1,start:b4,dpdown:h0.4,dpleft:h0.8,dpright:h0.2,dpup:h0.1,guide:b6,leftshoulder:b9,leftstick:b7,lefttrigger:a4,leftx:a0,lefty:a1,rightshoulder:b10,rightstick:b8,righttrigger:a5,rightx:a2,righty:a3,x:b2,y:b3,platform:android"

gamepad_test_mapping(index, mapping_string);

논의 기술
인덱스 설정할 게임 패드 인덱스 "슬롯"입니다.
색깔 사용할 맵 문자열입니다 (자세한 내용은 설명 참조).

예:

var mapping = gamepad_get_guid(global.padIndex) + "," + gamepad_get_description(global.padIndex);
var len = array_length(global.PadInstances);
for( i = 0; i < len; i += 2)
    {
    var left = global.PadInstances[i];
    var right = global.PadInstances[i+1];
    mapping += "," + left.sdlLabel + ":" + right.binding;
    }
gamepad_test_mapping(global.padIndex, mapping);

위의 코드는 여러 인스턴스를 반복하고 포함 된 다른 변수의 값을 사용하여 매핑 문자열을 만든 다음 지정된 슬롯 인덱스의 게임 패드에서 사용하도록 설정합니다.

sprite_replace

이 함수는 sprite_add () 와 거의 동일한 방식으로 작동하며, 가져 오는 스프라이트의 인덱스를 반환하는 대신 이전에 생성 된 스프라이트 인덱스를 덮어 씁니다. 이 기능을 사용하면이 같은 다른 기능을 사용하여 변수에 생성되고 저장되어있는 스프라이트 인덱스를 사용해야합니다 sprite_add () 또는 sprite_create_from_surface () , 또는 sprite_duplicate을 () 보다는 리소스 트리 스프라이트 자산을. 이 함수를 사용하여 게임 자산에서 스프라이트를 교체 할 수 있지만 그렇게하면 game_restart () 호출 여부에 관계없이 교체중인 스프라이트의 참조 ID가 손실되므로 권장하지 않습니다. 스프라이트 교체와 관계없이이 함수는 스프라이트에 대한 새로운 텍스처 페이지를 생성 하므로 렌더링에 필요한 텍스처 스왑 수를 늘려 성능에 부정적인 영향을 줄 수 있으므로이를 사용할 때주의를 기울여야합니다.

로드 할 이미지 파일은 항상 * .png 형식이어야하며 애니메이션 스프라이트로 변환 할 모든 이미지는 "스트립"형식이어야합니다 (아래 이미지 참조). 규칙 스프라이트 너비 = 스트립 너비 / 하위 이미지에 따라 지정된 하위 이미지 수로 분할됩니다.

위 이미지에서 볼 수 있듯이 스프라이트는 진한 자주색 배경에 배치되었으며 "removeback"인수를 true 로 설정하여 제거 할 수 있습니다. 스프라이트의 왼쪽 아래 픽셀에서 색상을 확인한 다음 제거 할 색상으로 사용합니다. 예를 들어 위의 이미지에서 왼쪽 아래 픽셀 색상이 녹색이면 스프라이트의 모든 녹색 부분이 제거되고 나머지 자주색 배경은 무시됩니다.

"removeback"옵션을 선택하면, "smooth"인수를 true 로 설정하여 GameMaker Studio 2가 스프라이트의 가장자리를 부드럽게 할 수도 있습니다. 이 모든 것은 스프라이트의 배경을 제거한 후 스프라이트 가장자리 주위에 반투명 테두리를 만드는 것입니다.

마지막으로 스프라이트의 x 및 y 원점 을 지정할 수도 있습니다. 이것은 스프라이트가 그것을 사용하는 인스턴스에 스프라이트가 "고정 된"지점이며, 항상 하나의 스프라이트 서브 이미지의 왼쪽 상단의 0,0을 기준으로 계산됩니다. 예를 들어,이 값이 (16,16)으로 설정된 32 x 32 픽셀 인 스프라이트는 중심에 원점이 있습니다.

기본적으로 모든 새 스프라이트에는 경계 상자가 자동으로 계산됩니다 (정확한 bbox는 스프라이트의 크기와 투명도에 따라 다름) . 그러나이를 사용자 정의 할 수 있습니다.이 경우 sprite_collision_mask () 함수도 사용해야합니다.

참고 : 선택한 대상 플랫폼에 따라 파일을 저장하고로드 할 수있는 위치가 제한됩니다. 자세한 내용은 파일 처리 를 참조하십시오.

참고 : HTML5 대상 게임에서이 기능을 사용하여 외부 서버에서 리소스를로드하는 경우 브라우저의 XSS 보호로 인해 여러 도메인에서 리소스를로드하려는 시도가 차단되어 다시 나타날 수 있습니다. 빈 결과.

 

구문:

sprite_replace(ind, fname, imgnumb, removeback, smooth, xorig, yorig);

논의 기술
산업 영구적으로 교체 할 스프라이트의 인덱스입니다.
fname 새 스프라이트를 만들 이미지의 파일 이름입니다.
imgnumb 스프라이트가 가로로 잘릴 프레임 수입니다. 1 = 하나의 단일 이미지 / * .gif
제거 배경색 (왼쪽 아래 픽셀)이있는 모든 픽셀을 투명하게 만들지 여부를 나타냅니다.
부드러운 가장자리를 부드럽게할지 여부를 나타냅니다.
코 리그 스프라이트의 왼쪽 위 모서리를 기준으로 한 원점의 x 좌표입니다.
yorig 스프라이트의 왼쪽 위 모서리를 기준으로 한 원점의 y 좌표입니다.

예:

sprite_replace(spr_banner, "gravemaker.png", 1, false, false, 0, 0);

위 코드는 "spr_banner"에 색인 된 이미지 자산을 외부 소스에서로드 된 다른 자산으로 대체합니다.

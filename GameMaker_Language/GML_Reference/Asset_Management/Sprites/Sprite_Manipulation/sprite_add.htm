sprite_add

이 기능을 사용하면로드 할 이미지 파일이 항상 * .png , * .gif , * .jpg / jpeg 또는 * .json 형식이어야하는 외부 소스에서 이미지를 스프라이트로 추가 할 수 있습니다 ( * json 파일은 Spine으로 만든 스켈레톤 애니메이션 스프라이트 를로드하는 데 사용됩니다. 이 함수는 새 스프라이트 인덱스를 반환하고 스프라이트와 관련된 모든 추가 코드에서 사용해야합니다. 이 함수를 HTML5와 함께 사용하거나 URL에서 이미지를 가져 오는 경우이 함수는 이미지로드 된 비동기 이벤트도 생성합니다. 자세한 내용 은 비동기 이벤트 페이지를 참조하십시오.

* .gif (아래 참조)를 제외한 애니메이션 스프라이트로 변환 될 모든 이미지는 "스트립"형식이어야하며 (아래 이미지 참조) 규칙에 따라 지정된 하위 이미지 수로 분할됩니다. 스프라이트 너비 = 스트립 너비 / 하위 이미지 .

위 이미지에서 볼 수 있듯이 스프라이트는 진한 자주색 배경에 배치되었으며 "removeback"인수를 true 로 설정하여 제거 할 수 있습니다. 스프라이트의 왼쪽 아래 픽셀에서 색상을 확인한 다음 제거 할 색상으로 사용합니다. 예를 들어 위의 이미지에서 왼쪽 아래 픽셀 색상이 녹색이면 스프라이트의 모든 녹색 부분이 제거되고 나머지 자주색 배경은 무시됩니다.

"removeback"옵션을 선택하면, "smooth"인수를 true 로 설정하여 GameMaker Studio 2가 스프라이트의 가장자리를 부드럽게 할 수도 있습니다. 이 모든 것은 스프라이트의 배경을 제거한 후 스프라이트 가장자리 주위에 반투명 테두리를 만드는 것입니다.

마지막으로 스프라이트의 x 및 y 원점 을 지정할 수도 있습니다. 이것은 스프라이트가 그것을 사용하는 인스턴스에 스프라이트가 "고정 된"지점이며, 항상 하나의 스프라이트 서브 이미지의 왼쪽 상단의 0,0을 기준으로 계산됩니다. 예를 들어,이 값이 (16,16)으로 설정된 32 x 32 픽셀 인 스프라이트는 중심에 원점이 있습니다.

* .gif 형식 이미지를 추가하는 경우 파일 이름과 이미지 번호 (첫 번째 프레임의 경우 1이어야 함) 만 지정하고 다른 모든 인수는 0으로 남겨 둘 수 있습니다 . GIF가로드됩니다 . * .json 스프라이트의 경우 이미지 번호는 1이어야하며 "removeback"및 "smooth"설정은 무시됩니다.

기본적으로 모든 새 스프라이트에는 경계 상자가 자동으로 계산됩니다 (정확한 경계 상자는 스프라이트의 크기와 투명도에 따라 다름) . 그러나이를 사용자 정의 할 수 있습니다.이 경우 sprite_collision_mask () 함수도 사용해야합니다.

이런 식으로 스프라이트를 추가 할 때 사용되는 메모리는 예상보다 클 것입니다. GameMaker 스튜디오 2는 텍스처 페이지로 스프라이트를 저장하기 때문입니다 및 총 메모리가 같은 스프라이트의 이미지 파일에 대해 예상되는 것보다 클 수 있도록 그것은 또한, GPU 메모리에 저장됩니다.

참고 : Sprite를 GameMaker Studio 2에로드 할 때 더 이상 필요하지 않은 경우 스프라이트를 다시 제거해야합니다 ( sprite_delete () 사용 ). 그렇지 않으면 메모리 누수로 인해 속도가 느려지고 게임이 충돌 할 수 있습니다.

참고 : iOS 의 경우 폴더에서 포함 된 파일을로드하는 경우 (예 : " Backgrounds / background1.png "와 같은 경로로) 폴더를 경로의 일부로 포함하지 않고 파일 이름 만 제공하면됩니다. 충분하다. 다른 모든 플랫폼에서는 파일 경로의 일부로 폴더를 제공해야합니다.

참고 : HTML5 의 경우 보안 서버에서 스프라이트를로드하는 경우 교차 출처 유형을 설정하고 절대 경로 대신 상대 경로를 사용해야합니다. 자세한 내용은 http_set_request_crossorigin () 을 참조하십시오.

 

구문:

sprite_add(fname, imgnumb, removeback, smooth, xorig, yorig);

논의 기술
fname 추가 할 파일의 이름 (문자열 파일 경로)입니다.
imgnum 하위 이미지 수를 나타내는 데 사용합니다 (단일 이미지 또는 * .gif의 경우 1 ).
제거 배경색 (왼쪽 아래 픽셀)이있는 모든 픽셀을 투명하게 만들지 여부를 나타냅니다.
부드러운 투명한 경우 가장자리를 매끄럽게할지 여부를 나타냅니다.
코 리그 스프라이트에서 원점의 ​​x 위치를 나타냅니다.
yorig 스프라이트에서 원점의 ​​y 위치를 나타냅니다.

 

보고:

Real (sprite ID value)

 

예:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);

위의 코드는 스프라이트를 게임에로드하고 해당 인덱스를 변수 "spr"에 저장합니다.

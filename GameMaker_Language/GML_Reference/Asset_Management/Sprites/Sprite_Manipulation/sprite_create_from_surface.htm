sprite_create_from_surface

이 함수를 사용하면 이전에 초기화 된 표면에서 스프라이트를 만들 수 있습니다 ( surface_create ()를 사용하여 표면을 만들 때 표면 인덱스 ID 값이 반환 됨 ). 함수에 입력 한 x 및 y 좌표는 게임 윈도우가 아닌 표면 의 (0,0) 위치 (왼쪽 위 모서리)를 기준으로해야합니다. width 및 height 인수는 픽셀 단위이며 사용할 표면 부분의 너비 및 높이를 정의합니다.

"removeback"인수를 true로 설정하면 스프라이트의 왼쪽 아래 픽셀에서 해당 색상을 확인한 다음 제거 할 색상으로 사용하여 스프라이트에서 색상을 제거합니다.

"removeback"옵션을 선택하면, "smooth"인수를 true 로 설정하여 GameMaker Studio 2가 스프라이트의 가장자리를 부드럽게 할 수도 있습니다. 이 모든 것은 스프라이트의 배경을 제거한 후 스프라이트 가장자리 주위에 반투명 테두리를 만드는 것입니다.

마지막으로 스프라이트의 x 및 y 원점 을 지정할 수도 있습니다. 이것은 스프라이트가 그것을 사용하는 인스턴스에 스프라이트가 "고정 된"지점이며, 항상 하나의 스프라이트 서브 이미지의 왼쪽 상단의 0,0을 기준으로 계산됩니다. 예를 들어,이 값이 (16,16)으로 설정된 32 x 32 픽셀 인 스프라이트는 중심에 원점이 있습니다.

기본적으로 모든 새 스프라이트에는 경계 상자가 자동으로 계산됩니다 (정확한 bbox는 스프라이트의 크기와 투명도에 따라 다름) . 그러나이를 사용자 정의 할 수 있습니다.이 경우 sprite_collision_mask () 함수도 사용해야합니다.

참고 : 이 방법으로 GameMaker Studio 2에서 스프라이트를 생성 할 때 더 이상 필요하지 않은 경우 스프라이트를 다시 제거해야합니다 ( sprite_delete () 사용 ). 그렇지 않으면 메모리 누수로 인해 속도가 느려지고 게임이 중단 될 수 있습니다.

 

통사론:

sprite_create_from_surface(index, x, y, w, h, removeback, smooth, xorig, yorig);

논의 기술
인덱스 만들 표면의 색인입니다.
엑스 복사 할 x 위치입니다.
와이 복사 할 y 위치입니다.
승 복사 할 영역의 너비입니다 (x 위치에서).
h 복사 할 영역의 높이입니다 (y 위치에서).
제거 배경색 (왼쪽 아래 픽셀)이있는 모든 픽셀을 투명하게 만들지 여부를 나타냅니다.
부드러운 가장자리를 부드럽게할지 여부를 나타냅니다.
코 리그 스프라이트에서 원점의 ​​x 위치를 나타냅니다.
yorig 스프라이트에서 원점의 ​​y 위치를 나타냅니다.

 

보고

Real (unique sprite ID value)

 

예:

var surf;
surf = surface_create(32, 32);
surface_set_target(surf);
draw_clear_alpha(c_black, 0);
draw_sprite(spr_Body, 0, 0, 0);
draw_sprite(spr_Clothes, 0, 0, 0);
draw_sprite(spr_Hair, 0, 0, 0);
spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
surface_reset_target();
surface_free(surf);

위의 코드는 표면을 작성하고 색인을 로컬 변수 "surf"에 저장합니다. 그런 다음 해당 표면을 대상으로 지정하고 지우고 서로 위에 여러 개의 스프라이트를 그립니다. 마지막으로 표면에 그려진 합성 이미지에서 새 스프라이트를 만들고 표면에 사용 된 메모리를 비우기 전에 해당 인덱스를 "spr_Custom"에 할당합니다.

sprite_collision_mask

이 기능을 사용하면 스프라이트가 가져야하는 충돌 마스크의 속성을 설정할 수 있습니다. 경계 상자 모드로 자동 (0) 또는 전체 이미지 (1)를 선택하면 개별 경계 상자 값을 0으로 설정할 수 있습니다. 그러나 사용자 정의 마스크 (2)의 경우이 값을 설정해야합니다. 다른 경계 상자 값은 항상 (0, 0) 위치로 간주되는 스프라이트의 왼쪽 상단 모서리 (x 및 y 원점에 관계없이)를 기준으로합니다.

마스크의 종류를 설정하면 마스크 자체의 일반적인 모양이 설정되지만, 사각형 마스크 이외의 다른 것들은 충돌을 해결할 때 더 많은 처리 능력을 필요로하며 결과적으로 성능이 저하됩니다. 일반적으로 꼭 필요한 경우 사각형 이외의 마스크 유형 만 사용해야합니다.

참고 :이 기능은 회전 된 사각형 충돌 마스크 종류를 허용하지 않습니다.

설정할 수있는 마스크 종류는 다음 상수 중 하나입니다.

일정한 기술
bboxkind_rectangular 직사각형 (비 회전) 직사각형 충돌 마스크 모양
bboxkind_ellipse 타원형 충돌 마스크 모양
bboxkind_diamond 다이아몬드 충돌 마스크 모양
bboxkind_precise 마스크가 주어진 공차 값에 따라 스프라이트의 불투명 픽셀을 준수하는 정확한 충돌 마스크 (아래 참조)

마지막으로, 허용 오차는 마스크가 얼마나 정확한지 정의하는 데 사용됩니다 ( "전체 이미지"마스크와 함께 사용하는 경우에는 효과가 없습니다). 허용 오차가 0이면 마스크가 투명도를 갖는 모든 단일 픽셀을 따릅니다. 다른 값은 픽셀의 투명도에 따라 충돌 마스크 주변을 이동시킵니다.

참고 :이 함수는 비트 맵 스프라이트에만 유용하며 SWF 또는 JSON (스파인) 스프라이트에서는 작동하지 않습니다.

참고 :이 기능은 추가 된 스프라이트 또는 복제 된 스프라이트에서만 작동하며 사전 제작 된 리소스 에서는 직접 작동 하지 않습니다 . sprite_duplicate () 함수를 사용하여 스프라이트를 복제 할 수 있습니다.

 

통사론:

sprite_collision_mask(ind, sepmasks, bboxmode, bbleft, bbtop, bbright, bbbottom, kind, tolerance);

논의 기술
산업 경계 상자를 설정할 스프라이트의 인덱스입니다.
격막 스프라이트의 각 하위 이미지에 대해 충돌 마스크를 만들지 ( true ) 또는 모두에 대해 하나의 마스크를 만들지 ( false ) 여부입니다.
bboxmode 사용할 경계 상자의 종류 0 = 자동, 1 = 전체 이미지, 2 = 사용자 정의
왼손잡이 경계 상자의 최대 왼쪽 위치입니다.
바베큐 경계 상자의 최대 상단 위치입니다.
브라이트 바운딩 박스의 최대 오른쪽 위치.
bbbottom 경계 상자의 최대 하단 위치입니다.
종류 마스크의 종류, 상수 (설명 표 참조).
공차 투명도 값의 공차를 나타냅니다 (0 = 공차 없음, 255 = 완전 공차).

 

보고

N/A

 

예:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);
sprite_collision_mask(spr, true, 1, 0, 0, 0, 0, 0, 0);

위 코드는 외부 소스에서 스프라이트를로드하고 변수 "spr"에 새 인덱스를 저장합니다. 그런 다음 코드는 새 스프라이트가 각 하위 이미지에 대해 정확한 충돌 마스크를 갖도록 설정합니다.

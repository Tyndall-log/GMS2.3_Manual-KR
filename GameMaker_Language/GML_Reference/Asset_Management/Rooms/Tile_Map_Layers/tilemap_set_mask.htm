tilemap_set_mask

이 기능은 단일 타일 맵의 타일 ​​비트 마스크 를 설정하는 데 사용할 수 있습니다. tilemap 요소 ID ( layer_tilemap_create ()를 사용하여 tilemap 요소를 만들 때 또는 layer_tilemap_get_id () 함수를 사용할 때 얻음 )와 마스크 값을 제공합니다. 예를 들어 16 x 16 타일의 아주 작은 타일 세트가 있다면 타일 세트에 총 256 개의 서로 다른 타일이 있으며 타일 데이터에 "8 비트의 색인 만 사용합니다" "(사용 가능한 19 비트 데이터). 이 비트들은 "마스킹"될 수 있고, 나머지 타일 데이터는 추가 값을 저장하는데 사용될 수있다.

초기 인덱스 마스크 값을 얻는 쉬운 방법은 타일 세트의 타일 수를 빼고 1을 빼는 것이므로 16x16 타일 세트에는 256 타일과 255의 마스크 값 (또는 16 진수로 $ ff)이 있습니다. 타일 ​​세트에는 2의 거듭 제곱이 아닌 여러 타일이 있고 마스크를 얻기 위해 가장 가까운 2의 거듭 제곱으로 반올림합니다. 예를 들어 20x20 인 타일 세트에는 400 개의 타일이 있으며 512까지 올림합니다. 따라서 마스크 인덱스 값 511 (16 진수 $ 1ff)을 얻습니다. 인덱스 마스크 외에도 일반적으로 타일에 지정된 플립, 회전 및 미러 값을 유지 하여 사용할 수있는 상수 가 몇 가지 있도록합니다. 이것들은 인덱스 마스크 값과 함께 "또는"처리되어 그 비트의 데이터를 보존 할 수 있습니다. 다음 상수를 사용할 수 있습니다 (사용 방법은 아래 예제 코드 참조).

 

일정한 기술
tile_rotate 타일 ​​데이터 블롭의 회전 비트를 설정 / 취득하는 데 사용됩니다.
tile_mirror 타일 ​​데이터 블롭의 미러 비트를 설정 / 취득하는 데 사용됩니다.
tile_flip 타일 ​​데이터 블롭의 플립 비트를 설정 / 취득하는 데 사용됩니다.
tile_index_mask 타일 ​​인덱스를 추출하기 위해 타일 데이터 블로 브로 "and"하는 특수 상수입니다.

 

Tilemap 비트 마스킹은 고급 기능으로, 타일 데이터 비트를 "블롭"으로 사용할 수 있습니다. 예를 들어, 사용자는 tiledata 구조에서 "솔리드"를 의미하기 위해 비트를 예약 한 후 빠른 충돌이 있는지 확인할 수 있습니다. 기본적으로 비트 맵은 타일 맵이 그려 질 때 타일 데이터에 대해 "및"처리됩니다 (따라서 다른 기능에는 영향을 미치지 않음). 마스크에없는 비트는 무시되지만 여전히 사용자가 자신의 목적을 위해 읽고 씁니다. 타일 ​​맵당 글로벌 마스크와 마스크라는 두 가지 마스크가 있으며,이 마스크는 GameMaker Studio 2에서 내부적으로 "및"연결되어 그리기에 사용됩니다.

 

구문:

tilemap_set_mask(tilemap_element_id, mask)

논의 기술
tilemap_element_id 마스크를 설정할 타일 맵 요소의 고유 ID 값
마스크 마스크 값

 

반환값:

N/A

 

예:

var lay_id = layer_get_id("Tiles_sky");
var map_id = layer_tilemap_get_id(lay_id);
var mask = tile_mirror | tile_flip | tile_rotate | 255;
tilemap_set_mask(map_id, mask);

위의 코드는 지정된 레이어에서 타일 맵 ID를 가져온 다음 타일 맵 비트 마스크를 8 비트로 설정하여 맵의 각 타일에 대한 플립, 미러 및 회전 값을 유지합니다.

network_set_config

이 기능을 사용하면 다른 네트워크 구성을 설정할 수 있습니다.

참고 :이 기능은 HTML5 대상에서 실행중인 프로젝트에서 사용될 때 작동하지 않습니다.

이러한 구성은 상수 로 제공되며 설정은 선택한 상수에 따라 다릅니다. 아래 표는 사용 가능한 상수와 해당 설정을 보여줍니다.

 

일정한 기술 환경
network_config_connect_timeout 연결 시간 초과 값 설정 밀리 초 단위의 값
network_config_use_non_blocking_socket GameMaker Studio 2에게 연결시 차단하지 말라고 알려주십시오. 0 = 차단, 1 = 차단 안 함
network_config_enable_reliable_udp 기존 UDP 소켓에 대해 "신뢰할 수있는 UDP"프로토콜을 활성화합니다 대상으로하는 소켓 ID
network_config_disable_reliable_udp 기존 UDP 양말에 대한 "신뢰할 수있는 UDP"프로토콜을 비활성화합니다. 대상으로하는 소켓 ID

첫 번째 옵션 ( network_config_connect_timeout )은 단순히 서버에 연결하기위한 시간 제한을 설정하지만 대기 시간과는 별도로 연결 동작을 변경하지는 않습니다. LAN 전용 게임이나 인터넷의 경우 4000 정도 정도로 너무 낮게 설정하지 않고 약 1000으로 유지하는 것이 좋습니다. 그렇지 않으면 게임이 임의로 연결되지 않을 수 있습니다. 패킷을 송수신하기위한 시간 초과 값을 설정하려면 network_set_timeout () 함수를 사용하십시오.

두 번째 옵션 ( network_config_use_non_blocking_socket )은 네트워크 연결 기능이 모두 즉시 socket_id를 반환 하지만 비동기 네트워크 이벤트를 수신 할 때까지 해당 소켓에서 보내거나받을 수 없음을 의미합니다 . 트리거 된 이벤트에는 "type"키가 network_type_non_blocking_connect로 설정됩니다 ( 네트워킹 비동기 이벤트 페이지에서 자세한 내용을 확인할 수 있음). GameMaker Studio 2는 하나의 응용 프로그램에서 블로킹과 비 블로킹을 혼합하여 지원하지 않으므로 전역 설정입니다.

세 번째 및 네 번째 옵션 ( network_config_enable_reliable_udp , network_config_disable_reliable_udp )은 GameMaker Studio 2의 신뢰할 수있는 UDP 프로토콜을 활성화 또는 비활성화하기위한 것이므로 UDP 연결에서만 사용해야 합니다 . 이 경우 "신뢰할 수있는"것은 연결의 양쪽 끝에서 활성화되면 GameMaker Studio 2가 패킷이 올바르게 도착했는지 확인하고 도착하지 않은 패킷을 다시 보내려고 시도한다는 것을 의미합니다. 당신의 패킷에 전송 순서대로 도착합니다). 안정적인 소켓 유형을 사용하는 경우, 모든 UDP 패킷에 12 바이트 헤더가 추가되는데, 여기에는 GameMaker Studio 2에서 패킷의 오류를 확인하고 누락 된 패킷을 다시 전송하는 데 필요한 정보가 들어 있습니다. 이 옵션을 사용할 때 다음 사항에 유의하십시오.

  • 프로토콜은 데이터를 보내기 전에 양쪽에서 활성화 / 비활성화되어야하며 옵션을 활성화 / 비활성화하기 전에 보내거나받은 데이터는 영향을받지 않습니다.
  • 서버 측과 클라이언트 측 모두에서 올바르게 관리되는 한 지정된 소켓 ID에 대해 여러 번 프로토콜을 활성화 또는 비활성화 할 수 있습니다.
  • 하나는 신뢰할 수있는 UDP를 사용하도록 설정하고 다른 하나는 신뢰할 수있는 UDP를 사용하지 않도록 설정하는 두 개의 개별 UDP 소켓을 가질 수 있습니다.

 

구문:

network_set_config(config_value, setting);

논의 기술
config_value 설정할 구성 값 상수 (아래에 나열 됨).
환경 구성 값의 설정

 

반환값:

String

 

예:

network_set_config(network_config_connect_timeout, 1000);
network_set_config(network_config_use_non_blocking_socket, 1);

위의 코드는 네트워크 연결 시간 초과를 1000ms로 설정하고 GameMaker Studio 2가 연결시 차단하지 않도록 지시합니다.

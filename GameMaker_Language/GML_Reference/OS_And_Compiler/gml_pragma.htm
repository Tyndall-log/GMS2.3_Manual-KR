gml_pragma

gml_pragma 함수는 주어진 대상이 코드를 컴파일하는 방법에 영향을 미치며 프로젝트의 최종 컴파일을 더욱 최적화하기 위해 다른 명령으로 호출해야합니다. 이 명령은 게임이 컴파일되기 전에 효과적으로 사전 처리 되므로 함수를 프로젝트의 어느 곳에 나 배치 할 수 있으며 게임이 완전히 컴파일되기 전에 처리됩니다. 사용 가능한 명령은 다음과 같습니다.

  • " forceinline "- 스크립트 함수 에서 "forceinline"pragma를 사용하여 함수를 호출하면 YYC는 참조가 아닌 인라인 함수로 프로젝트를 컴파일합니다. 이것은 추가 처리 향상을 가져다 주지만 특히 인라인 코드가 크거나 여러 다른 장소에서 사용되거나 크게 증가하는 경우 최종 실행 파일 크기를 부 풀릴 수 있으므로 사용할 때주의를 기울여야합니다. 컴파일 시간 이 pragma는 YYC를 사용하여 빌드 할 때만 유효합니다 .
  • " global ", " [gml code] "- "global"pragma를 사용하면 게임의 첫 번째 방이 실행되기 전에 컴파일 시간에 전역 범위에서 문자열 형식의 일부 GML 코드를 호출 할 수 있습니다. 예를 들면 다음과 같습니다. gml_pragma ( "global", "Init ()"); 게임의 첫 번째 방이 실행되기 전에 스크립트 함수 "Init"을 호출합니다. 두 번째 인수로 제공된 GML 은 컴파일 시간 상수 여야하며, 이 pragma를 사용하여 인스턴스를 작성하거나 룸 (또는 룸의 모든 항목)이 작동해야하는 조작을 수행 할 수는 없습니다.
  • " PNGCrush "- "PNGCrush"pragma는 생성 ​​된 각 텍스처에 PNGCrush 프로그램을 사용합니다. 게임을 컴파일하는 데 걸리는 시간이 상당히 길어질 수 있으므로 최종 파일 크기를 크게 절약 할 수는 있지만 항상 원하지는 않습니다.
  • " Texgroup.Scale ", " [TextureGroupName] ", " [Scale Divisor] "- "Texgroup.Scale"은 컴파일시 주어진 텍스처 그룹을 스케일합니다. 여기에 추가로 두 개의 인수를 추가해야합니다 : 스케일링 할 텍스처 그룹의 이름 (문자열) 인 "[TexGroupName]"및 제수 인 "[Scale Divisor]"(문자열) 스케일링에 사용하려는 경우 : gml_pragma ( "Texgroup.Scale", "level1", "2"); "level1"텍스처 그룹에있는 모든 텍스처의 절반이됩니다.
  • " UnityBuild ", " [enable / disable] "-이 pragma로 함수를 호출하고 enable / disable 인수를 true 로 설정하면 컴파일시 프로젝트의 모든 .cpp 파일을 단일 파일로 축소합니다. 예를 들어 다음과 같이 모든 것을 빌드하는 데 사용합니다. gml_pragma ( "UnityBuild", "true"); 유니티 빌드의 이점은 빌드가 빠르지 만 매번 전체 빌드를 수행한다는 것입니다. 코드의 한 부분을 변경하더라도 캐시 된 파일을 사용하지 않고 모든 것을 다시 빌드합니다. 다른 빌드 ( YYC 전용 )를 위해 호출 할 수 있지만 YYC를 사용하여 Xbox One 내보내기를 위해 특별히 추가되었습니다. 유니티 빌드에 대한 자세한 내용은 여기 를 참조 하십시오 .

참고 : gml_pragma 함수의 첫 번째 인수는 변수가 아닌 컴파일 타임 문자열 상수 여야합니다 .
